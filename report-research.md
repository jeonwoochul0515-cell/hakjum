# 학점나비 보고서 형태 서비스 전환 리서치

> 조사일: 2026-04-01
> 목적: 현재 분절된 탐색 플로우를 "종합 보고서" 중심으로 재설계 + 결제 시점 최적화

---

## 1. 현재 문제 진단

### 1-1. "이 대학교의 상세 데이터가 아직 없어요" 원인

**표시 조건** (`UniversityDetailStep.tsx:234`):
```
info === undefined && stats === undefined → 메시지 표시
```

**원인 우선순위:**

| 순위 | 원인 | 영향 범위 | 해결 난이도 |
|:---:|------|:---:|:---:|
| 1 | **DATA_GO_KR_API_KEY 미설정** | 전체 대학 | 낮음 (환경변수 설정) |
| 2 | **학교명 불일치** (커리어넷 vs data.go.kr) | 특정 대학 | 중간 (퍼지 매칭) |
| 3 | **API 오류 침묵 실패** (`.catch(() => {})`) | 간헐적 | 중간 (로깅 추가) |
| 4 | **data.go.kr에 학과 데이터 미등록** | 소규모/신생 학과 | 높음 (데이터 부재) |
| 5 | **비동기 로드 타이밍** (데이터 도착 전 렌더링) | 일시적 | 낮음 (로딩 상태 구분) |

**즉시 해결 방안:**
1. Cloudflare Pages 대시보드에서 `DATA_GO_KR_API_KEY` 환경변수 설정
2. "데이터 없음" vs "로딩 중" 구분 UI 추가
3. API 오류 시 콘솔 로깅 추가

### 1-2. 현재 플로우의 구조적 문제

```
학교선택 → 관심사 → AI학과추천 → 학과상세 → 대학선택 → 대학상세 → 과목추천
   ①          ②         ③          ④          ⑤         ⑥         ⑦

문제:
- 7단계가 분절되어 있어 "결국 뭘 받는 건지" 불명확
- 중간 탐색(④⑤⑥)에서 "왜 여기서 돈을 내야 하지?" 저항 발생
- 최종 결과(⑦)가 한 화면의 과목 리스트 → 보고서로서의 가치감 부족
```

## 2. 보고서 형태 서비스 벤치마크

### 2-1. 경쟁사 결제 모델 분석

| 서비스 | 무료 범위 | 결제 트리거 | 가격 | 전환 핵심 |
|--------|----------|-------------|------|----------|
| **진학사** | 대학별 합격등급 (한 줄) | "상세 분석 보기" 클릭 | 건당 3~5천 / 시즌 2~4만 | 매몰 비용 (성적 입력 완료) |
| **유웨이** | 기본 배치표 위치 | 정밀 분석 클릭 | 유사 | 불안감 (내 위치가 애매) |
| **16personalities** | 유형 결과 + 기본 설명 | "프리미엄 프로필" | $30~50 | 감정적 공감 ("나를 정확히 묘사") |
| **와이즈멘토** | 없음 (B2B) | 학교 단체 구매 | 2~5만 | B2B 채널 |
| **입시 컨설팅** | 없음 | 1:1 상담 | 50~300만 | 전문가 신뢰 |

### 2-2. 핵심 인사이트

> **"돈을 내고 받을 가치가 있는 것"이 명확해야 한다.**
> 현재 학점나비는 7단계를 거쳐야 결과를 받는데, 중간에 결제를 넣으면 "왜 여기서?" 저항이 생김.
> 보고서 형태로 바꾸면 **"이 보고서 전체를 보려면"**이라는 자연스러운 결제 근거가 됨.

## 3. 보고서 중심 플로우 재설계

### 3-1. 제안 플로우: 7단계 → 4단계

```
[1단계] 입력 (2~3분)                    ← 무료
  ├─ 학교 선택
  ├─ 관심사/희망 진로 입력
  └─ (선택) 희망 대학 입력

[2단계] AI 분석 중 (15~30초)             ← 무료
  ├─ "AI가 127개 학과를 분석하고 있습니다..."
  ├─ "교과이수기준 데이터를 매칭하고 있습니다..."
  └─ "맞춤 로드맵을 생성하고 있습니다..."
  → 프로그레스 + 메시지로 "가치 있는 작업 중" 인지

[3단계] 보고서 미리보기 + 결제           ← 결제 시점
  ├─ 무료 공개: 프로필 요약 + 추천 학과 TOP 3 + 충족률 %
  ├─ 유료 잠금: 상세 분석, 로드맵, 입시 전략 (블러 처리)
  └─ CTA: "전체 보고서 확인하기 — 4,900원"

[4단계] 전체 보고서 열람                 ← 유료
  ├─ 모든 섹션 공개
  ├─ PDF 다운로드
  └─ 공유 링크 생성
```

### 3-2. 중간 탐색의 통합

| 현재 (분절) | 보고서 내 통합 |
|------------|---------------|
| 학과 상세 (④) | → 섹션 3 "학과 상세 분석" |
| 대학 선택 (⑤) | → 입력 단계에서 "희망 대학" 또는 AI 자동 매칭 |
| 대학 상세 (⑥) | → 섹션 5 "입시 전략 분석" + 섹션 9 "경쟁 데이터" |
| 과목 추천 (⑦) | → 섹션 7 "AI 맞춤 과목 추천" + 섹션 8 "3년 로드맵" |

**단, 자유 탐색 기능은 별도 유지** — 무료 사용자의 재방문을 유도하는 역할

## 4. 보고서 구조 설계

### 4-1. 전체 구성 (10개 섹션, 10~15페이지)

| # | 섹션 | 내용 | 무료/유료 |
|---|------|------|:---------:|
| 1 | **학생 프로필** | 학교, 학년, 관심 분야, 희망 진로 | 무료 |
| 2 | **AI 추천 학과 TOP 10** | 적합도 점수 + 한 줄 사유 (리스트만) | 무료 (TOP 3) |
| 3 | **학과 상세 분석** | 커리큘럼, 졸업 후 진로, AI 근거 | **유료** |
| 4 | **목표 대학 매칭** | 추천 학과 개설 대학 + 합격 난이도 | 상위 3개 무료 |
| 5 | **입시 전략 분석** | 수시 vs 정시 적합도, 전형별 전략 | **유료** |
| 6 | **교과이수기준 충족률** | 목표 대학 기준 vs 현재 과목 비교표 | **유료 (핵심)** |
| 7 | **AI 맞춤 과목 추천** | 필수/강추/고려/선택 4단계 티어링 | 필수만 무료 |
| 8 | **3년 과목 이수 로드맵** | 1~3학년 학기별 추천 과목 배치 | **유료 (핵심)** |
| 9 | **경쟁 데이터** | 경쟁률, 커트라인, 합격자 내신 분포 | **유료** |
| 10 | **종합 의견 + 액션 플랜** | AI 코멘트 + 당장 해야 할 3가지 | **유료** |

### 4-2. 무료 vs 유료 경계 설계

**무료 공개 원칙**: "결과가 있다"는 것은 보여주되, "해석과 해결책"은 잠금

```
무료: "교과이수기준 충족률: 62%"        ← 숫자만
유료: "미충족 과목 3개: 미적분, 물리학I, 화학I → 2학년 1학기에 수강 필요"  ← 해석+액션

무료: "추천 학과 1위: 컴퓨터공학 (적합도 92점)"  ← 리스트만
유료: "컴퓨터공학과를 위해 정보과학 + 미적분 조합이 최적인 이유..."  ← 근거

무료: "3년 로드맵이 생성되었습니다" + 블러 썸네일
유료: 전체 로드맵 열람
```

### 4-3. 차별화 포인트 (경쟁사에 없는 것)

1. **교과이수기준 충족률** — 2028 대입부터 필수, 학점나비만 보유
2. **학교별 개설과목 기반 현실적 추천** — NEIS 연동, 경쟁사 전무
3. **3년 로드맵** — 컨설팅에서만 제공하던 것을 AI로 자동 생성
4. **입시결과 연동 역산 추천** — 커트라인 기반 과목 역추천, 시장 유일

## 5. 결제 시점 최적화

### 5-1. 전환율을 높이는 전략

**1) 매몰 비용 활용**
- 학교 선택 + 관심사 입력에 2~3분 투자 → "여기까지 했는데 안 보면 아깝다"

**2) 감정적 반응 활용**
- AI 추천 학과 TOP 3을 무료 공개 → "오, 나한테 이런 학과가 맞구나!" 공감
- 충족률 % 공개 → "62%밖에 안 되네... 뭘 해야 하지?" 불안 → 해결책은 유료

**3) 시각적 블러 처리**
- 보고서 전체 레이아웃은 보이되, 유료 섹션은 블러
- "이미 생성된 보고서가 기다리고 있다"는 느낌

**4) 사회적 증거**
- "이 보고서를 받은 학생 87%가 과목 선택을 변경했습니다"
- "오늘 142명이 보고서를 다운로드했습니다"

**5) 긴급성**
- "수강신청 마감까지 D-14"
- "2학년 과목 선택은 1학년 11~12월에 결정됩니다"

### 5-2. 가격 구조 (기존 유지, 포지셔닝 변경)

| 플랜 | 가격 | 내용 | 포지셔닝 |
|------|------|------|----------|
| 무료 체험 | 0원 | 하루 3회 AI 추천 + 보고서 미리보기 | "내 결과를 확인해보세요" |
| 보고서 1회 | 4,900원 | 전체 보고서 + PDF | "입시 컨설팅의 1/100 가격" |
| 연간 올인원 | 7,900원 | 무제한 재생성 + PDF | "고교 3년 내내 활용하세요" |

**가치 프레이밍**: "입시 컨설팅 50~300만원 → 학점나비 4,900원"

## 6. 보고서 포맷

### 6-1. 웹 보고서 (메인)

- 단일 긴 페이지 (원페이저) — 스크롤하며 읽음
- 상단 목차(앵커 링크)로 섹션 이동
- 카드 형태 섹션 구분
- 인터랙티브: 대학 클릭→상세 펼침, 과목 클릭→AI 설명 팝업
- 하단 고정: "PDF 다운로드" / "공유하기"

### 6-2. PDF 다운로드 (부가)

- A4 세로, 10~15페이지
- 커버: 학생 이름, 생성일, 학교명, 학점나비 로고
- 인쇄 친화적 (흑백 구분 가능 색상)
- 마지막 페이지: 면책 조항 + QR코드(웹 보고서 링크)
- 학점나비에 이미 `jspdf` + `html-to-image` 설치됨 → 기존 `pdf-report.ts` 패턴 활용

### 6-3. 공유 기능

- **카카오톡 공유** — 학부모 전달이 핵심 사용 시나리오
- **고유 URL** — 읽기 전용, 유효기간 30일
- **교사용 요약** — 불필요한 정보 숨기고 핵심만 표시
