## 1. Executive Summary

## 2. 2028 대입제도 변화와 서비스 영향

> 출처: 교육부 2023.12.27 「2028 대학입시제도 개편 확정안」

### 2-1. 확정된 변화 사항

#### (1) 통합형 수능 (2027.11 시행, 2028학년도 적용)

| 영역 | 현행 (2027학년도까지) | 2028학년도 |
|------|---------------------|-----------|
| 국어 | 화법과작문/언어와매체 **선택** | **통합 출제** (선택 폐지) |
| 수학 | 확률과통계/미적분/기하 **선택** | **통합 출제** (공통 범위) |
| 영어 | 절대평가 | 절대평가 유지 |
| 탐구 | 사회/과학 17과목 중 최대 2 선택 | 사회/과학 **9과목** 중 최대 2 선택 |
| 한국사 | 절대평가 | 절대평가 유지 |

**팩트 체크 결과**: 수학 출제범위에 미적분II(심화미적분) 포함 여부, 탐구 과목 수 최종 확정 등은 아직 세부 조정 중

#### (2) 내신 평가 체계 전환

| 항목 | 현행 | 2025 고1~ |
|------|------|----------|
| 등급 체계 | 9등급 상대평가 | **5등급 상대평가 + 성취평가(A-E) 병행** |
| 1등급 비율 | 상위 4% | **상위 10%** |
| 등급 분포 | 4/7/12/17/20/17/12/7/4% | **10/24/32/24/10%** |
| 석차/석차백분율 | 기재 | **폐지** |
| 이수 단위 | 단위 수 | **학점 수** |

**팩트 체크 결과 (중요한 수정)**: 에이전트 보고서에서 "절대평가 성격의 5등급제"로 기술했으나, 정확히는 **5등급 상대평가와 성취도(A-E) 절대평가가 병행** 기재되는 방식. 대학마다 어떤 지표를 반영할지는 자율 결정.

#### (3) 고교학점제 전면 시행

- 2025년 고1부터 적용 — 192학점 이수, 성취평가제 확대
- 학생이 진로에 따라 과목 선택 → 어떤 과목을 이수했는지가 대입의 핵심 변수
- 대학은 모집단위별 **교과이수기준(필수/권장 과목)을 사전 공개** 의무

### 2-2. 주요 대학 교과이수기준 현황

> 출처: 대교협 2025.12, 2026.02 자료집 (학점나비 코드베이스에 이미 반영)

| 대학 | 이공계열 필수 | 의약계열 필수 | 인문계열 |
|------|-------------|-------------|---------|
| 서울대 | 미적분+기하+물리학I | 미적분+화학I+생명과학I | 제2외국어/한문 1과목 |
| 연세대 | 미적분+물리학I+화학I | 미적분+화학I+생명과학I+II | 사회교과 2과목 |
| 고려대 | 미적분+과학I 2과목 | 미적분+화학I+생명과학I | 사회교과 2과목 |
| 성균관대 | 미적분+과학I 2과목 | - | 사회교과 2과목 |
| 한양대 | 미적분+물리학I/화학I | 미적분+화학I+생명과학I | - |
| 경희대 | 미적분+과학I 1과목 | 미적분+화학I+생명과학I | 권장만 |
| 부산대 | 미적분+과학I 1과목 | 화학I+생명과학I | 국어심화1+사회1 |

**공통 트렌드**:
- 이공계: 미적분이 사실상 **필수**, 과학I 최소 1~2과목
- 의약계: 화학I+생명과학I **공통 필수**, 과학II 이수가 유리
- 인문계: 사회교과 다양 이수, 서울대는 제2외국어 필수
- 상경계: 미적분+확률과통계 핵심, 경제 과목 권장

### 2-3. 학점나비 서비스 영향

#### 긍정적 영향 — 서비스 가치 급상승

1. **과목 선택의 중요성 극대화**: "어떤 과목을 들어야 하는가"가 모든 고1의 핵심 고민 → 학점나비의 핵심 가치
2. **교과이수기준 의무화**: 대학이 요구 과목을 공개 → 학점나비의 "역방향 연결" 기능 필수화
3. **NEIS 연동의 가치**: "우리 학교에 이 과목이 있는가?"를 확인해주는 유일한 서비스

#### 대응 필요 사항

| 우선순위 | 과제 | 설명 |
|:---:|------|------|
| **상** | 내신 5등급제 반영 | AI 프롬프트/입시 전략에서 9등급→5등급+성취도 기반으로 업데이트 |
| **상** | 2028 통합형 수능 반영 | 정시 전략 안내를 2028 체제에 맞게 수정 |
| **중** | 교과이수기준 확대 | 현재 14개 대학 → 30개 이상 확대 |
| **중** | 과목명 매핑 검증 | 2025 고1 NEIS 데이터 반영 시 2022 교육과정 과목명 매칭 정확도 확인 |
| **중** | 공동교육과정 안내 | 학교에 필수 과목 미개설 시 대안 경로(클러스터, 온라인) 안내 |
| **하** | 3년 로드맵 추천 | 현재 단일 학년 → 1~3학년 과목 이수 계획 로드맵 |

### 2-4. 확정 vs 미확정 정리

| 확정 | 미확정/유동적 |
|------|-------------|
| 2022 개정 교육과정 2025 고1 적용 | 수능 수학 출제범위 최종 (미적분II 포함 여부) |
| 통합형 수능 (선택과목 폐지) | 수능 탐구 과목 수 최종 확정 |
| 5등급 상대평가 + 성취평가 병행 | 5등급 내신의 대학별 변별 방식 |
| 교과이수기준 사전 공개 의무화 | 수능 최저학력기준 설정 방식 |
| 석차등급/석차백분율 폐지 | 일부 대학 교과이수기준 세부 조정 |

## 3. 경쟁 서비스 분석

### 3-1. 주요 경쟁사 비교

#### 진학사 (jinhak.com)

| 항목 | 내용 |
|------|------|
| 핵심 서비스 | 수시/정시 합격 예측 (성적→합격 가능 대학), 원서접수 대행 |
| 데이터 | 20년+ 입시결과 축적, 실제 합격자 데이터 |
| 가격 | 합격예측 3.9~8.9만원, 원서접수 건당 ~5,000원 |
| 시장 점유율 | 원서접수 ~45% (유웨이와 양강) |
| **과목 추천** | **"골라줘! 선택과목" 서비스 존재** — 학과별 권장과목 목록 제공 |
| **한계** | NEIS 학교 개설과목 연동 없음, AI 개인화 추천 아닌 정적 목록, 학교별 맞춤 불가 |

> **팩트 체크 수정**: 에이전트 보고서에서 "진학사에 과목 추천 없음"이라 했으나, 실제로는 hijinhak.jinhak.com에서 "골라줘! 선택과목" 서비스를 제공 중. 단, 학과→권장과목 정적 매핑 수준이며, "내 학교 개설과목과 교차 분석" 기능은 없음.

### 3-3. 차별화 기회와 위협

#### 학점나비만의 해자(Moat)

1. **역방향 연결**: "목표 대학 → 유리한 과목 추천" — 시장 유일
2. **NEIS 실시간 연동**: "내 학교에 이 과목이 있는가?" — 경쟁사 전무
3. **압도적 가격 경쟁력**: 컨설팅 대비 1/25~1/250 가격
4. **타이밍**: 고교학점제 1기(2025 고1) → 2028 대입까지 선점 기회

#### 위협 요인

| 위협 | 심각도 | 대응 |
|------|:------:|------|
| 진학사가 NEIS 연동 + AI 추천 추가 | 중 | 이미 기술적 선점, 속도가 관건 |
| 교육부/교육청 공식 서비스 출시 | 중 | 공공 서비스는 UX/AI 품질에서 열세 |
| 입시결과 데이터 확보 어려움 | **상** | PDF 수집(단기) + 공공데이터 신청(장기) 병행 |
| 대형 에듀테크 시장 진입 | 하 | 뤼이드/콴다는 학습 영역에 집중, 다른 도메인 |

## 4. 입시결과 데이터 확보 전략

### 4-1. 현재 보유 데이터 vs 필요 데이터

#### 이미 있는 것

| API/데이터 | 내용 | 수준 |
|-----------|------|------|
| `tn_pubr_public_univ_major_api` | 학과명, 입학정원, 졸업자수, 주요교과목 | 대학 단위 |
| `tn_pubr_public_univ_reg_amt_api` | 평균등록금, 입학금, 설립형태 | 대학 단위 |
| 대학알리미 8개 지표 | 경쟁률, 취업률, 충원율, 중퇴율 등 | **대학 전체 평균** |

#### 없는 것 (학생이 가장 원하는 데이터)

| 데이터 | 예시 | 공식 API |
|--------|------|:---:|

### 4-3. 방안 1: 대학어디가 크롤링 — 비추천

**법적 불가 사유**:
- 대학어디가 이용약관: "서비스를 이용하여 얻은 정보를 가공, 판매하는 행위 등 서비스에 게재된 자료를 **상업적으로 사용할 수 없다**"
- 저작권법 DB 제작자 권리: 7.8만건 체계적 복제 → 상당 부분 복제에 해당
- 대법원 2021도1533 판결(크롤링 무죄)은 약관 제한이 없는 경우에 한정 → 대학어디가에는 불리

### 4-4. 방안 2: 공공데이터 제공 신청 — 장기 전략

#### 신청 절차

| 단계 | 내용 | 소요기간 |
|------|------|----------|
| 1 | data.go.kr "공공데이터 제공신청"에서 온라인 신청 | 즉시 |
| 2 | 해당 기관(교육부/대교협) 이송 및 검토 | - |

#### 성공 가능성: 50~70%

**유리한 점**:
- 대교협은 이미 대학알리미 데이터를 data.go.kr API로 제공 중 (선례)
- 학점나비의 "고교학점제 지원" 목적은 교육 공익적 명분이 강함
- 공공데이터법에 의한 제공 의무, 거부 시 분쟁조정 가능

**불리한 점**:
- 커트라인은 각 대학 제출 민감 데이터 → 제3자 권리 주장 가능
- 대교협이 자신을 공공기관이 아닌 민간 협의체로 주장할 가능성
- 행정 속도 느림

### 4-5. 방안 3: 대학 입학처 공개 PDF 수집 — 최우선 실행

#### 검증된 사실

| 대학 | 입시결과 공개 방식 | 확인 |
|------|-------------------|:---:|
| 서울대 | admission.snu.ac.kr/materials/stats/result 웹 공개 | O |
| 연세대 | 입학처 웹사이트 합격결과 조회 (전형별) | O |
| 고려대 | 입학처 영상/PDF 안내 | O |
| 기타 주요 대학 | 대부분 입학처에서 PDF/웹 형태로 공개 | O |

#### 왜 최선인가

- **100% 합법**: 대학이 자발적으로 공개한 데이터, 출처 표기만 하면 됨
- **즉시 시작 가능**: 법적 검토 불필요
- **상위 30개 대학으로 학생 관심의 70%+ 커버** (파레토 법칙)
- **Claude API로 PDF 파싱 자동화** 가능

#### 대상 대학 30개교

**서울 상위**: 서울대, 연세대, 고려대, 성균관대, 한양대, 서강대, 중앙대, 경희대, 한국외대, 서울시립대, 건국대, 동국대, 홍익대, 숙명여대, 이화여대

**특수목적**: KAIST, POSTECH, GIST, UNIST, 서울교대

**거점국립대**: 부산대, 경북대, 전남대, 충남대, 전북대, 충북대, 강원대, 제주대

**기타**: 연세대(미래), 고려대(세종)

### 4-6. 데이터 스키마

```typescript
interface AdmissionResult {
  university: string;       // 대학명
  major: string;            // 학과명
  year: number;             // 2025
  admissionType: string;    // "학생부교과" | "학생부종합" | "논술" | "정시"
  period: 'susi' | 'jeongsi';
  recruited: number;        // 모집인원
  applied: number;          // 지원자수
  competitionRate: number;  // 경쟁률
  cutline: {
    avg: number;            // 평균 등급 or 점수
    percentile70: number;   // 70%컷
    min: number;            // 최저
  };
  supplementaryOrder: number | null;  // 충원 순번
}
```

**저장**: `data/admission-results/{year}-{period}.json` → 빌드 시 `public/data/`로 복사
**갱신**: 연 2회 (수시결과 2~3월, 정시결과 4~5월)

## 5. PDF 파싱 기술 스택

### 5-1. 라이브러리 비교

| 라이브러리 | 표 추출 | 한국어 | Node.js | CF Workers | 비용 |
|-----------|:------:|:------:|:------:|:----------:|------|
| **pdf-parse** (설치됨) | X | O | O | O | 무료 |
| **pdfjs-dist** (설치됨) | 수동구현 | O | O | O | 무료 |
| pdf2json | 수동구현 | O | O | O | 무료 |
| tabula-js | **O** | O | O | **X (Java)** | 무료 |
| Camelot | **O** | O | X (Python) | X | 무료 |

### 5-2. AI 기반 파싱 비교

| 방식 | 한국어 표 정확도 | 구현 난이도 | 30개 대학 비용/회 | 기존 인프라 |
|------|:---:|:---:|:---:|:---:|
| **Claude API (PDF 직접)** | **90~95%** | **낮** | **$1.5~$15** | **이미 연동** |
| GPT-4o Vision | 85~90% | 중 | $3~$30 | 추가 필요 |
| Google Document AI | 90~95% | 중~상 | $0~$20 | 추가 필요 |

> Claude API PDF 지원 확인: platform.claude.com/docs/ko/build-with-claude/pdf-support에서 PDF 직접 입력 정식 지원 확인됨

### 5-3. 최종 권장안: pdfjs-dist + Claude API 하이브리드

```
PDF 다운로드 → pdfjs-dist로 텍스트+좌표 추출 → Claude API로 구조화 → JSON
                (무료, 이미 설치)              (이미 연동, 비용 극저)
```

**이유**:
1. pdfjs-dist와 Claude API **모두 이미 프로젝트에 설치/연동**
2. Cloudflare Workers 호환 (Java/Python 의존성 없음)
3. 대학마다 다른 PDF 양식에 AI가 유연하게 대응
4. 비용 극히 저렴: **연간 $3~$15**
5. 기존 `sync.ts` 패턴(크롤링→파싱→KV 캐시) 확장 가능

**Fallback**: pdfjs-dist 텍스트 추출 실패(스캔 PDF) → Claude Vision(이미지) fallback

### 5-4. 검증/QA 파이프라인 (필수)

1. **스키마 검증**: zod로 AdmissionResult 인터페이스 검증
2. **범위 검증**: 경쟁률 0~200, 내신등급 1.0~9.0 범위 체크
3. **전년도 비교**: 급격한 변동 감지 (경쟁률 50% 이상 변동 시 플래그)
4. **학과 수 검증**: 알려진 학과 수와 비교
5. **수동 검토 큐**: 검증 실패 항목 관리자 알림

### 5-5. 구현 구조

```
scripts/admission-pdf-sync.ts   — 로컬: PDF 다운로드 + 파싱 + 구조화
functions/api/admission/results.ts — API: KV 캐시 데이터 제공
data/admission-results/*.json     — 정적 JSON 저장
```

**예상 개발 기간**: 3~5일 (파싱 로직 + 검증 + 30개 대학 URL 수집)

## 6. 법적 고려사항

### 6-1. 리스크 수준 요약

| 항목 | 리스크 | 근거 |
|------|:------:|------|
| 숫자/통계 데이터 추출 | **낮음** | 사실 데이터는 저작권 비보호 (저작권법 제2조) |
| PDF 레이아웃/표현 복제 | 중~높음 | 편집저작물 침해 가능 (저작권법 제6조) |
| DB 제작자 권리 | 중간 | 대량 복제 시 해당 가능 (저작권법 제93조) |
| 부정경쟁방지법 | 낮음~중간 | 대학 공개 자료이므로 상대적 안전 |
| 개인정보보호법 | **매우 낮음** | 집계 통계만 활용 시 무관 |
| 대학 이의 제기 | 낮음~중간 | 출처 표기로 완화 가능 |

### 6-2. 핵심 법적 원칙

1. **사실/데이터 ≠ 저작물**: 경쟁률 3.5:1, 커트라인 285점 등 숫자는 객관적 사실 → 저작권 보호 대상 아님
2. **표현 ≠ 사실**: PDF의 표 디자인/레이아웃은 편집저작물일 수 있으나, 데이터 값 자체를 추출하여 **자체 포맷으로 재구성**하면 문제없음
3. **공개 자료**: 대학이 자발적으로 일반에 공개한 자료이므로 접근 자체는 적법

```
  - "적정": 건국대 컴공 (커트라인 3.1) — 현재 가능
  - "안정": 세종대 컴공 (커트라인 3.8) — 여유
```

```
→ "정보과학/물리학II 선택하면 컴공 + 반도체공학 두 학과 모두 지원 가능"
```
