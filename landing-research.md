# 학점나비 랜딩페이지 리디자인 리서치

> 조사일: 2026-04-01
> 목적: 사용자 동선 혼란 해결 + 서비스 가치 직관적 전달을 위한 랜딩페이지 리디자인

---

## 1. 현재 문제 진단

### 1-1. 핵심 문제

| 문제 | 상세 |
|------|------|
| **"뭘 해주는 서비스인지" 전달 부족** | Hero가 공포 소구("잘못된 과목 선택이 입시를 바꿉니다")에 집중, 서비스 기능 설명 약함 |
| **동선 안내 위치가 너무 아래** | "이용 방법 3단계"가 8번째 섹션(하단)에 위치 — 대부분 스크롤 안 함 |
| **실제 화면 프리뷰 없음** | "이런 결과를 받을 수 있다"는 시각적 증거가 전혀 없음 |
| **플로우 구조 혼란** | 실제 7~8단계인데 "3단계"로 요약 → 체감 괴리 |
| **탐색 구간이 일직선 Wizard** | 학과→대학 탐색은 자유롭게 돌아다녀야 하는데 일직선으로 강제 |

### 1-2. 현재 섹션 순서 (문제점 표시)

```
1. Hero: 공포 소구 헤드라인              ← "뭘 해주는지" 안 보임
2. 걱정 카드 로테이션 (5개)              ← 불안 가중
3. 핵심 수치 (전국/NEIS/대교협/30초)
4. 첫 CTA                              ← 스크롤 2~3회 후에야 등장
5. 대상별 CTA (학부모/교사)
6. 핵심 가치 ("진로 확실하지 않아도 OK")
7. Before/After 비교 (텍스트)
8. 이용 방법 3단계                       ← 여기까지 스크롤하는 사람 적음
9. 후기 (4개 시나리오)
10. 신뢰 배지
11. FAQ (5개)
12. 최종 CTA
```

### 1-3. 미사용 자산 발견

- `src/components/layout/StepIndicator.tsx` — 3단계 인디케이터("내 학교", "내 꿈", "내 과목")가 이미 만들어져 있으나 **실제 FlowPage에서 사용되지 않고 있음**

## 2. 리서치 결과: 베스트 프랙티스

### 2-1. 고전환 랜딩페이지의 정보 구조 (RMCA 프레임워크)

```
1. Relevance (적합성)  → "이 서비스가 나를 위한 것인지" — Hero 헤드라인
2. Mechanism (메커니즘) → "어떻게 가치를 만들어주는지" — How it works
3. Confidence (신뢰)   → "증거와 리스크 해소" — 사회적 증거, 데이터 출처
4. Action (행동)       → "하나의 명확한 다음 단계" — CTA
```

**현재 학점나비**: R(공포) → R(공포) → R(수치) → A → A → R(공감) → R(비교) → **M(이용방법)** → C(후기) → C(신뢰) → C(FAQ) → A

**문제**: Mechanism이 너무 뒤에 있음. 사용자가 "어떻게 작동하는지" 이해하기 전에 CTA를 만남.

### 2-2. 에듀테크 랜딩 트렌드 2026

| 트렌드 | 설명 | 학점나비 적용 |
|--------|------|-------------|
| **인터랙티브 데모** | 스크린샷 대신 실제 워크플로우 애니메이션 | 추천 결과 프리뷰 |
| **즉시 체험** (Duolingo 방식) | 가입 없이 바로 핵심 기능 체험 | 랜딩에서 관심사 바로 입력 |
| **Sticky Bottom CTA** | 스크롤 시 하단 고정 CTA → 전환율 15~25% 증가 | 추가 필요 |
| **점진적 정보 수집** | 처음엔 최소 정보만 → 나머지는 과정 중 수집 | 이미 적용됨 (좋음) |
| **매 fold마다 CTA** | 주요 구간마다 CTA 배치 → 단일 CTA 대비 20~35% 증가 | 현재 3곳 → 유지/강화 |

### 2-3. Z세대(고등학생) UX 핵심

| 원칙 | 적용 |
|------|------|
| **3초 규칙** | 첫 화면에서 3초 안에: (1)뭐하는 서비스? (2)나한테 왜 필요? (3)어떻게 시작? |
| **공포 < 탐색** | Z세대는 이미 충분히 불안 → 서비스가 불안 가중하면 이탈 |
| **모바일 90%+** | 고등학생 트래픽 90% 이상 모바일 → 모바일 퍼스트 필수 |
| **짧은 주의력** | 한 화면에 하나의 메시지만 |
| **게이미피케이션** | 과도하면 진지함 훼손, 가벼운 마이크로 보상 정도 |

### 2-4. 동선 명확화 패턴

#### 현재 플로우의 구조적 문제

```
입력 구간        탐색 구간                    결과 구간
(Wizard 적합)    (자유 탐색 적합인데 Wizard)   (산출물)
───────────     ─────────────────────────    ──────────
학교 → 관심사 → 학과목록 → 학과상세 → 대학목록 → 대학상세 → 과목추천
                ↑___________________________↑
                이 구간에서 "어디야?" 혼란 발생
```

#### 권장 구조: 3-Phase 모델

```
Phase 1: 내 정보        Phase 2: AI 탐색          Phase 3: 맞춤 결과
━━━━━━━━━━━━━━         ━━━━━━━━━━━━━━━━         ━━━━━━━━━━━━━━━
학교 선택               학과 탐색 허브             과목 추천 결과
관심사 입력             (학과목록/상세/대학을       입시 전략
(적성검사 선택)          탭/모달로 자유 전환)       PDF 리포트
```

- Phase 2를 하나의 "탐색 허브" 페이지로 만들어, 학과 목록↔상세↔대학을 일직선이 아닌 **탭/패널/모달**로 전환
- 기존 StepIndicator 3단계 ("내 학교", "내 꿈", "내 과목")와 자연스럽게 대응

## 3. 랜딩페이지 리디자인 안

### 3-1. 권장 섹션 순서

```
1. Hero — 가치 제안 + 즉시 CTA
2. How it Works — 3단계 + 실제 화면 프리뷰
3. CTA #1
4. 핵심 수치 + 신뢰 배지 (통합)
5. 인터랙티브 미니 체험 (관심사 입력 → 샘플 결과)
6. 대상별 진입 (학생/학부모/교사)
7. 후기
8. FAQ
9. 최종 CTA
+ Sticky Bottom CTA (스크롤 시 하단 고정)
```

### 3-2. 각 섹션 상세

#### 섹션 1: Hero (Above the Fold)

**변경 전:**
```
"잘못된 과목 선택 한 번이 대학 입시를 바꿉니다"
(공포 소구, 서비스가 뭔지 안 보임)
```

**변경 후:**
```
[한 줄 설명] AI 과목 추천 서비스
[메인 헤드라인] 내 꿈에 딱 맞는 과목,
               30초면 찾아드려요
[서브 헤드라인] 학교 개설과목 + 대학 입시기준을
               AI가 교차 분석해서 최적 과목을 추천합니다
[CTA] 지금 바로 시작하기
[마이크로카피] 무료 · 회원가입 없음 · 즉시 결과
```

**핵심 변화:**
- 공포 소구 → 혜택 중심 가치 제안
- "이 서비스가 뭘 하는지" 3초 안에 전달
- CTA가 Above the fold에 위치

#### 섹션 2: How it Works (3단계 + 화면 프리뷰)

```
━━━ 이렇게 진행돼요 ━━━

[1] 관심사를 알려주세요        [2] AI가 학과를 찾아줘요      [3] 맞춤 과목을 받으세요
    "의사가 되고 싶어요"           의예과, 약학과, 간호학과         필수: 화학I, 생명과학I
    학교도 골라주세요              각 대학 입시결과까지            강추: 미적분, 물리학I
                                                               → 서울대 교과이수기준 90% 충족

    [입력 화면 프리뷰]            [학과 카드 프리뷰]            [결과 화면 프리뷰]
```

**핵심:**
- 각 단계에 **실제 서비스 화면의 축소 프리뷰** 삽입
- "이런 화면이 나오고, 이런 결과를 받는구나"를 시각적으로 전달
- 기존 8번째 위치 → 2번째로 이동

#### 섹션 3: CTA #1

```
[CTA] 내 맞춤 과목 찾기
[마이크로카피] 전국 고등학교 지원 · NEIS 실시간 연동
```

#### 섹션 4: 핵심 수치 + 신뢰 (통합)

```
[전국 고등학교]  [NEIS 실시간]  [대교협 공식]  [개인정보 수집 없음]
   데이터 연동      과목 연동       입시기준         안심 이용

데이터 출처: 커리어넷 · NEIS 공공데이터 · 대교협 2028 교과이수기준
AI 엔진: Claude AI (Anthropic)
```

기존 핵심 수치(앵커링)와 신뢰 배지를 하나로 합침.

#### 섹션 5: 인터랙티브 미니 체험

```
━━━ 지금 바로 체험해보세요 ━━━

[입력 필드] 어떤 일에 관심 있어? (예: 프로그래머, 의사, 디자이너...)
[제출 버튼] AI 추천 받기 →

→ 제출 시 /flow로 이동, interest가 이미 채워진 상태
```

**핵심:**
- Duolingo 방식 — 설명 대신 바로 체험
- 랜딩에서 관심사를 입력하면 school-select로 이동 (또는 interest 저장 후 school-select)
- 전환율 극대화

#### 섹션 6: 대상별 진입

```
[학생]                    [학부모]                  [교사·학원]
"내 진로에 맞는            "자녀의 과목 선택          "학생별 맞춤
 과목을 찾고 싶어요"        도와주고 싶어요"           진로지도가 필요해요"
[시작하기]                [시작하기]                [자세히 보기]
```

기존 대상별 CTA를 유지하되, 더 깔끔하게.

#### 섹션 7: 후기

기존 4개 시나리오 유지 (적절함).

#### 섹션 8: FAQ

기존 5개 항목 유지. 필요시 축소.

#### 섹션 9: 최종 CTA

```
━━━ 아직 고민 중이라면 ━━━
무료로 시작하세요. 30초면 됩니다.

[CTA] 무료로 맞춤 과목 찾기
```

#### Sticky Bottom CTA (신규)

```
┌─────────────────────────────────┐
│  내 맞춤 과목 찾기  [시작하기 →] │  ← 스크롤 시 하단에 고정
└─────────────────────────────────┘
```

- Hero가 뷰포트를 벗어나면 나타남
- 슬림하게 (높이 48px 정도)
- 전환율 15~25% 증가 효과

### 3-3. 삭제/이동 대상

| 기존 섹션 | 처리 |
|----------|------|
| 걱정 카드 로테이션 (5개) | **삭제** — 공포 가중, Z세대에 역효과 |
| Before/After 비교 | **삭제** — 텍스트 나열로는 효과 약함, How it Works가 대체 |
| 핵심 가치 ("진로 확실하지 않아도 OK") | **FAQ로 이동** 또는 How it Works 서브 메시지로 통합 |

## 4. 플로우 동선 개선안

### 4-1. FlowHeader 개선

**현재:** 8단계 프로그레스 바 + `text-[10px]` 스텝 라벨
**권장:** 3-Phase 인디케이터 (이미 만들어진 StepIndicator.tsx 활용)

```
[● 내 정보 ━━━] → [○ AI 탐색 ───] → [○ 맞춤 과목 ───]
```

- Phase 1: school-select, interest-input, aptitude-*
- Phase 2: major-results, major-detail, university-list, university-detail
- Phase 3: subject-match

### 4-2. 각 단계에 "왜" 한 줄 추가

| 단계 | 현재 | 추가할 한 줄 |
|------|------|-------------|
| school-select | "먼저 학교를 골라볼까요?" | "학교를 알려주면 **실제 개설과목** 중에서 추천해요" |
| interest-input | "어떤 꿈을 꾸고 있나요?" | "관심사를 알려주면 AI가 **맞는 학과**를 찾아줘요" |
| major-results | (없음) | "AI가 찾은 학과예요. 궁금한 학과를 눌러보세요" |
| university-detail | (없음) | "이 대학의 입시결과를 참고해서 과목을 추천해드려요" |

### 4-3. 탐색 허브화 (중기 과제)

Phase 2의 4개 스텝(major-results → major-detail → university-list → university-detail)을 하나의 탐색 허브로 통합:
- 학과 목록이 메인 뷰
- 학과 클릭 → 슬라이드 패널로 상세 표시
- 대학 클릭 → 같은 패널 내에서 전환
- 뒤로가기 반복 없이 자유 탐색

### P0: 즉시 개선 (1~2일)

| # | 작업 | 파일 | 효과 |

| 4 | Sticky Bottom CTA 추가 | HomePage.tsx | 전환율 15~25% 증가 |
| 5 | 섹션 순서 재배치 (RMCA 순서) | HomePage.tsx | 정보 구조 최적화 |

| # | 작업 | 파일 | 효과 |
|---|------|------|------|
| 6 | 3단계에 실제 화면 프리뷰 추가 | HomePage.tsx | "뭘 받는지" 시각적 전달 |

| # | 작업 | 파일 | 효과 |
